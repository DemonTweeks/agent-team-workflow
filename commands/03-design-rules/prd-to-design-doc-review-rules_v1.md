## 适用范围

- 本规则仅用于检查按 `prd-to-design-doc-mapping-rules_v1.md` 生成的设计文档。
- 若文档并非按该规则生成，需在报告中标注「检查适配性不足」。

## 用户输入

```text
$ARGUMENTS
```

输入可以是：
- 待检查设计文档路径（如 `{模块名}_设计文档.md` 或合订本）
- （可选）对应 PRD 路径（用于映射对账）
- （可选）对应 `01-接口文档.md` 路径（用于交叉引用对账）
- （可选）功能清单路径（用于模块边界校验）

## 指令说明

你需要生成一份「需求存量设计文档完整性检查报告」，验证文档是否严格符合 `prd-to-design-doc-mapping-rules_v1.md`。

检查必须覆盖六个维度：
1. **文档结构完整性**：文档级、模块级、方法块级结构是否齐全
2. **PRD 映射一致性**：PRD 功能点 ↔ Controller 方法双向可追溯、命名一致、状态完整
3. **方法块完整性**：10 个必选子节、表格列、Mermaid 图、证据锚点
4. **严格事实模式**：EXTRACTED 占比、证据锚点、CONFLICT/UNKNOWN_RUNTIME 处理
5. **质量门禁**：覆盖、链路、数据、图表、映射、严格事实六大门禁
6. **交叉引用一致性**：设计文档 ↔ 接口文档 ↔ PRD 三处同源

---

## 检查任务

### 第一步：文档级结构完整性检查

#### 1.1 必备文档级块检查（核心）

在第一个具体方法块之前，必须包含以下块：

| 序号 | 块名 | 最低要求 | 缺失时 |
|------|------|----------|--------|
| 1 | 标题 | `# {服务名} 需求存量设计文档` | **阻断** |
| 2 | 修订记录 | 版本、日期、作者、变更摘要（至少表头） | **主要** |
| 3 | 服务业务域概览 | 服务职责、核心业务能力、技术栈、关键依赖 | **主要** |
| 4 | **PRD 覆盖清单** | 表格：序号、模块名、PRD 文件名、功能点数量、Controller 数量、映射状态 | **阻断** |
| 5 | 模块业务地图 | 模块清单、职责简述、模块间依赖关系（文字或 Mermaid） | **主要** |
| 6 | **方法索引** | 表格：序号、模块、Controller#method、中文名、对应 PRD 功能点、涉及表、事务、异步调用 | **阻断** |

**检查方法**：逐项检查是否存在；内容为空且未标注「待补充」→ **严重错误**。

#### 1.2 PRD 覆盖清单完整性检查

**检查项**：
- 是否列出本文档覆盖的全部模块
- 每个模块的 PRD 文件名是否与目录中实际文件一致
- 功能点数量是否与 PRD 中 BR-XX 数量一致（允许 ±1 误差，须说明）
- Controller 数量是否为实际扫描结果
- 映射状态是否为 `完整` / `部分` / `待补充` 之一

**未满足时**：
- 缺失模块 → **严重错误**
- 功能点数量明显不符且无说明 → **错误**
- 映射状态全为「完整」但正文存在 `PRD_UNMAPPED` / `BLOCKED` → **严重错误**

#### 1.3 方法索引完整性检查

**检查项**：
- 是否包含文档中所有 Controller 方法
- 每个方法是否标注对应 PRD 功能点（BR-XX 或 `TECH_METHOD`）
- 涉及表、事务、异步调用三列是否非空（无则写「无」）

**未满足时**：
- 方法缺失 → **严重错误**
- 对应 PRD 功能点全空 → **错误**

---

### 第二步：模块级结构完整性检查

#### 2.1 模块章节边界检查

每个模块必须使用一个二级标题 `## {序号}. {模块名}`，且必须包含：

| 子块 | 最低要求 | 缺失时 |
|------|----------|--------|
| 模块职责引用 | `> **模块职责**`、`> **边界说明**`、`> **关键实体**`、`> **对应 PRD**` | **主要** |
| `{序号}.1 模块职责与边界` | 模块职责叙述、与相邻模块分界 | **主要** |
| `{序号}.2 PRD 功能点映射表` | 表格：BR 编号、功能点名称、对应 Controller、对应方法、映射状态、备注 | **阻断** |
| `{序号}.3 方法级业务链路` | 至少一个方法块（四级标题） | **阻断** |
| `{序号}.4 模块规则清单汇总` | 汇总本模块全部业务规则（或写「见各方法子节」） | **主要** |
| `{序号}.5 模块枚举与状态机总表` | 汇总本模块全部枚举与状态机（或写「无」） | **主要** |
| `{序号}.6 模块数据库表影响总表` | 汇总本模块全部表影响（或写「无」） | **主要** |

**检查方法**：逐项检查；缺失 4/5/6 但方法子节完整 → **错误**；缺失 2 或 3 → **严重错误**。

#### 2.2 PRD 功能点映射表检查

**检查项**：
- 是否列出该模块 PRD 中全部 BR-XX 功能点
- 每个 BR-XX 是否映射到至少一个 Controller 方法，或标记为 `PRD_UNMAPPED`
- 映射状态是否为 `MAPPED` / `PARTIAL` / `PRD_UNMAPPED` / `TECH_METHOD` / `BLOCKED` 之一
- 存在 `PARTIAL` / `BLOCKED` 时，备注列是否说明原因

**未满足时**：
- BR-XX 缺失 → **严重错误**
- 映射状态非法 → **错误**
- `PARTIAL` / `BLOCKED` 无备注 → **错误**

---

### 第三步：方法块级完整性检查

#### 3.1 方法块边界检查

每个方法必须使用一个四级标题 `#### {序号}.3.n {Controller#method} — {方法中文名称}`，且必须包含 10 个必选子节：

| 子节 | 标题语义 | 最低要求 | 缺失时 |
|------|----------|----------|--------|
| 1 | 方法标识 | 表格或列表覆盖 8 个子项（模块名、Controller、方法名、路由、对应 PRD 功能点、PRD 文档引用、接口文档引用、业务说明） | **阻断** |
| 2 | 主链路（流程图） | Mermaid `flowchart TD` 或 `flowchart LR`，含输入校验→权限校验→业务编排→数据操作→对外调用→返回组装 | **阻断** |
| 3 | 时序图 | Mermaid `sequenceDiagram`，含调用方、Controller、Service、Domain、Repository、DB、外部服务 | **阻断** |
| 4 | 分支链路 | 表格：序号、分支条件、触发阈值、分支动作、分支返回、对应 PRD 规则、代码位置 | **主要** |
| 5 | 异常链路 | 表格：异常类型、抛出点、捕获点、转换规则、返回码/HTTP状态、对应 PRD 异常场景、日志/告警、代码位置 | **主要** |
| 6 | 业务规则分类 | 表格：规则类别、规则说明、触发条件、校验/执行位置、失败处理、对应 PRD 规则、证据锚点 | **主要** |
| 7 | 枚举与状态机 | 7.1 枚举清单表 + 7.2 状态机（Mermaid `stateDiagram-v2` 或表格）；无状态机时写「本方法不涉及状态机」 | **主要** |
| 8 | 数据库与表影响 | 8.1 方法级表影响表 + 8.2 SQL 明细（关键操作）+ 8.3 表级汇总；无涉库时写「本方法不涉及数据库操作」 | **主要** |
| 9 | 下游依赖与稳定性 | 表格：依赖类型、依赖目标、调用目的、超时设置、重试策略、熔断/降级、补偿机制、对应 PRD 集成依赖、代码位置 | **主要** |
| 10 | 方法链路追踪 | 文本级调用链（主链路 + 异常链），含类型标注、特殊标记（TX/RPC/MQ/Cache/分支/异常） | **主要** |

**检查方法**：
- 每个方法块逐项核对 10 个子节
- 子节标题语义一致即可，允许前缀编号（如「1. 方法标识」）
- 缺失整节 → 按上表等级判定
- 子节存在但表格列明显缺失（如方法标识缺「对应 PRD 功能点」）→ **错误**

#### 3.2 方法标识子节检查

必须覆盖 8 个子项：

| 子项 | 检查点 | 缺失时 |
|------|--------|--------|
| 模块名 | 所属模块 | **错误** |
| Controller | 类全名或简名 | **错误** |
| 方法名 | Java/Go 方法名 | **错误** |
| 路由 | HTTP Method + Path | **错误** |
| **对应 PRD 功能点** | BR-XX 编号及名称 | **严重错误** |
| **PRD 文档引用** | `{模块}_业务需求文档.md` 章节锚点 | **严重错误** |
| 接口文档引用 | `01-接口文档.md` 章节锚点 | **错误** |
| 业务说明 | 一句话业务职责 | **错误** |

**特殊判定**：
- 标记为 `TECH_METHOD` 时，「对应 PRD 功能点」可写 `TECH_METHOD`，但须说明技术支撑原因
- 标记为 `BLOCKED` 时，须在「业务说明」或备注中说明阻断原因

#### 3.3 主链路流程图检查

**检查项**：
- 是否为 Mermaid `flowchart TD` 或 `flowchart LR`
- 节点是否覆盖：输入校验 → 权限校验 → 业务编排 → 数据操作 → 对外调用 → 返回组装
- 每个节点是否标注代码位置（类名#方法名）
- 条件判断是否为菱形节点并标注判断条件
- 事务边界是否用虚线框标注（`TX` 开始 / `TX` 结束 / `TX` 回滚）
- **PRD 映射标注**：流程图节点旁是否标注 `[BR-XX]`

**未满足时**：
- 非 Mermaid 流程图 → **严重错误**
- 缺少输入校验/权限校验/返回组装任一节点 → **错误**
- 节点无代码位置标注 → **错误**
- 无 PRD 映射标注 → **错误**
- 事务边界存在但未标注 → **错误**

#### 3.4 时序图检查

**检查项**：
- 是否为 Mermaid `sequenceDiagram`
- 参与者是否包含：调用方、Controller、Service、Domain、Repository、DB、外部服务/MQ/缓存（至少 5 个）
- 消息是否标注方法名和关键参数
- 异步调用是否用虚线箭头（`-->>`）
- 事务边界是否用 `rect` 包裹并标注传播行为
- 异常返回是否标注（`Note over X: Exception`）
- **PRD 映射标注**：时序图注释中是否标注 `[BR-XX]`

**未满足时**：
- 非 Mermaid 时序图 → **严重错误**
- 参与者少于 5 个 → **错误**
- 消息无方法名 → **错误**
- 无 PRD 映射标注 → **错误**

#### 3.5 分支链路检查

**检查项**：
- 是否为表格形式
- 表格列是否齐全：序号、分支条件、触发阈值/条件表达式、分支动作、分支返回、对应 PRD 规则、代码位置
- 每个分支是否标注代码位置
- **对应 PRD 规则**列是否非空（无对应规则时写「无」）

**未满足时**：
- 非表格形式 → **错误**
- 列缺失 → **错误**
- 分支无代码位置 → **错误**
- 无 PRD 规则标注 → **错误**

#### 3.6 异常链路检查

**检查项**：
- 是否为表格形式
- 表格列是否齐全：异常类型、抛出点、捕获点、转换规则、返回码/HTTP状态、对应 PRD 异常场景、日志/告警、代码位置
- 是否区分业务异常与系统异常
- 是否标注事务回滚触发情况
- **对应 PRD 异常场景**列是否非空

**未满足时**：
- 非表格形式 → **错误**
- 列缺失 → **错误**
- 未区分业务/系统异常 → **错误**
- 无 PRD 异常场景标注 → **错误**

#### 3.7 业务规则分类检查

**检查项**：
- 是否为表格形式
- 表格列是否齐全：规则类别、规则说明、触发条件、校验/执行位置、失败处理、对应 PRD 规则、证据锚点
- 规则类别是否覆盖：准入规则、校验规则、流转规则、并发规则、幂等规则、时效规则、审计规则、补偿规则（至少出现一次，无则写 N/A）
- 每个规则是否有证据锚点（file_path + symbol）
- **对应 PRD 规则**列是否非空

**未满足时**：
- 非表格形式 → **错误**
- 列缺失 → **错误**
- 规则无证据锚点 → **错误**
- 无 PRD 规则标注 → **错误**

#### 3.8 枚举与状态机检查

**检查项**：
- 7.1 枚举清单是否为表格：枚举名、所属类、用途、全部值与含义、对应 PRD 枚举、代码位置
- 7.2 状态机：是否为 Mermaid `stateDiagram-v2` 或表格
- 状态机表格列是否齐全：当前状态、允许下一状态、触发事件、守卫条件、失败回滚策略、对应 PRD 规则、代码位置
- **对应 PRD 枚举/规则**列是否非空

**未满足时**：
- 枚举清单非表格 → **错误**
- 状态机无 Mermaid 图且无表格 → **错误**
- 无 PRD 标注 → **错误**

#### 3.9 数据库与表影响检查

**检查项**：
- 8.1 方法级表影响表列是否齐全：database、schema、table、operation、where_keys、write_fields、join_tables、index_or_unique_dependency、transaction_scope、对应 PRD 数据需求、代码位置
- 8.2 SQL 明细：关键操作是否展开 SQL 或伪 SQL
- 8.3 表级汇总表列是否齐全：表名、业务用途、上游接口、核心字段语义、逻辑删除字段、乐观锁字段、高风险操作、对应 PRD 数据需求
- **对应 PRD 数据需求**列是否非空

**未满足时**：
- 表列缺失 → **错误**
- 涉库方法无 SQL 明细（复杂查询/批量更新/无 where 更新）→ **错误**
- 无 PRD 数据需求标注 → **错误**

#### 3.10 下游依赖与稳定性检查

**检查项**：
- 是否为表格形式
- 表格列是否齐全：依赖类型、依赖目标、调用目的、超时设置、重试策略、熔断/降级、补偿机制、对应 PRD 集成依赖、代码位置
- 每个依赖是否标注失败后的处理策略
- 异步调用是否标注消息可靠性
- **对应 PRD 集成依赖**列是否非空

**未满足时**：
- 列缺失 → **错误**
- 依赖无失败策略 → **错误**
- 无 PRD 集成依赖标注 → **错误**

#### 3.11 方法链路追踪检查

**检查项**：
- 是否提供文本级调用链（主链路 + 异常链）
- 每个节点是否标注类型（Controller/Service/Domain/Repository/DB/External）
- 是否标注特殊标记（TX/RPC/MQ/Cache/分支/异常）
- **是否标注对应 PRD 功能点编号**

**未满足时**：
- 无主链路或异常链 → **错误**
- 节点无类型标注 → **错误**
- 无 PRD 功能点标注 → **错误**

---

### 第四步：PRD 映射一致性检查

#### 4.1 双向可追溯检查

**检查项**：
- **PRD → 设计文档**：每个 BR-XX 在 PRD 中是否能在设计文档中找到对应方法或 `PRD_UNMAPPED` 标记
- **设计文档 → PRD**：每个方法在「方法标识」中是否标注了 BR-XX 或 `TECH_METHOD`
- **设计文档 ↔ 接口文档**：每个方法是否引用 `01-接口文档.md` 对应章节

**检查方法**：
- 读取 PRD 文档中全部 BR-XX 编号
- 读取设计文档中「PRD 功能点映射表」和「方法索引」
- 读取接口文档索引（如有）
- 核对三者交叉引用是否完整

**未满足时**：
- PRD 功能点无映射且无 `PRD_UNMAPPED` 说明 → **严重错误**
- 方法无 BR-XX 且无 `TECH_METHOD` 说明 → **严重错误**
- 方法未引用接口文档 → **错误**

#### 4.2 命名一致性检查

**检查项**：
- PRD 中的功能点名称与设计文档中的方法中文名称是否语义一致
- 同一 BR-XX 在流程图、时序图、分支链路、异常链路中的名称是否一致

**未满足时**：
- 命名明显矛盾（如 PRD 写「创建订单」，设计文档写「查询库存」且映射同一 BR-XX）→ **严重错误**
- 同一 BR-XX 在不同子节中名称不一致 → **错误**

#### 4.3 映射状态一致性检查

**检查项**：
- 「PRD 功能点映射表」中的映射状态是否与正文方法块中的标注一致
- 标记为 `MAPPED` 的方法是否确实在正文中存在对应方法块
- 标记为 `PRD_UNMAPPED` 的 BR-XX 是否在正文中无对应方法块
- 标记为 `BLOCKED` 的方法是否在正文中说明阻断原因

**未满足时**：
- 映射状态与正文矛盾 → **严重错误**
- `BLOCKED` 无原因说明 → **错误**

---

### 第五步：严格事实模式检查

#### 5.1 EXTRACTED 结论占比检查

**检查项**：
- 主文档正文（10 个子节内容）中，`EXTRACTED` 结论占比是否为 100%
- `INFERRED` / `AMBIGUOUS` 是否仅出现在「候选/待确认」区块或附录中

**检查方法**：
- 抽样检查方法标识、业务规则分类、数据库与表影响、下游依赖等子节
- 检查证据锚点的 `confidence` 字段

**未满足时**：
- 正文出现 `INFERRED` / `AMBIGUOUS` 且未在附录中说明 → **严重错误**
- 无证据锚点的方法块占比 > 5% → **严重错误**

#### 5.2 证据锚点完整性检查

**检查项**：
- 每个方法块的关键结论是否附证据锚点
- 证据锚点是否包含：file_path、symbol、evidence_type
- 是否有证据摘要说明该证据支持了什么结论

**未满足时**：
- 关键结论无证据锚点 → **错误**
- 证据锚点缺少 file_path / symbol / evidence_type 任一 → **错误**

#### 5.3 CONFLICT / UNKNOWN_RUNTIME 处理检查

**检查项**：
- 证据冲突时是否标记 `CONFLICT` 并并列展示
- PRD 与代码冲突时是否标注 `PRD_CODE_CONFLICT`
- 运行时不可见内容是否标记 `UNKNOWN_RUNTIME`
- `CONFLICT` / `UNKNOWN_RUNTIME` / `PRD_UNMAPPED` 是否全部入附录或质量报告

**未满足时**：
- 冲突未标注 `CONFLICT` → **严重错误**
- PRD 与代码冲突未标注 `PRD_CODE_CONFLICT` → **严重错误**
- `UNKNOWN_RUNTIME` 未给出缺失证据类型和建议补证路径 → **错误**

#### 5.4 防发散禁令检查

**检查项**：
- 是否使用「通常/一般/大概率/推测」等措辞
- 是否推断外部服务内部逻辑
- 是否将命名相似直接视为业务等价
- 是否根据历史经验补全状态机、补偿逻辑、权限规则
- 是否在无 SQL/ORM 证据时写数据库表关系
- 是否将 PRD 描述直接当作代码实现而未验证

**检查方法**：
- 全文关键词扫描（通常、一般、大概率、推测、可能、应该）
- 抽样检查数据库表影响子节的证据来源

**未满足时**：
- 使用禁用措辞代替证据 → **错误**
- 无 SQL/ORM 证据写表关系 → **严重错误**
- PRD 描述未经验证直接作为代码结论 → **严重错误**

---

### 第六步：质量门禁检查

#### 6.1 覆盖门禁

| 检查项 | 要求 | 检查方法 | 未满足时 |
|--------|------|----------|----------|
| Controller 方法覆盖率 | 100% | 对比代码中实际 Controller 方法数与设计文档中方法块数 | **严重错误** |
| 模块归类覆盖率 | >= 98% | 检查是否有方法落入 `misc` 且无说明 | **错误** |
| 接口文档共存率 | 每个方法同时存在于接口文档 | 抽查方法索引中的接口文档引用 | **错误** |
| 业务规则分类完整率 | >= 98% | 检查是否有方法缺失业务规则分类子节 | **错误** |
| **PRD 功能点映射率** | **>= 95%** | 计算 `MAPPED` / (总 BR-XX 数) | **严重错误**（若 < 90%）/ **错误**（若 90%~95%） |

#### 6.2 链路门禁

| 检查项 | 要求 | 检查方法 | 未满足时 |
|--------|------|----------|----------|
| 主链完整性 | 每个方法至少 1 条完整主链到数据层或外部依赖 | 检查方法链路追踪子节 | **严重错误** |
| 分支链路 | 有分支必须有分支链路 | 检查分支链路子节 | **错误** |
| 异常链路 | 有异常必须有异常链路 | 检查异常链路子节 | **错误** |
| 事务边界 | 有事务必须标明事务边界和传播行为 | 检查主链路流程图和时序图 | **错误** |
| 状态机 | 涉状态机必须有状态图或流转矩阵 | 检查枚举与状态机子节 | **错误** |

#### 6.3 数据门禁

| 检查项 | 要求 | 检查方法 | 未满足时 |
|--------|------|----------|----------|
| 涉库标注 | 涉库方法必须标 database/schema/table/operation/关键字段 | 检查数据库与表影响子节 | **错误** |
| 枚举标注 | 使用枚举的方法必须标枚举值语义 | 检查枚举与状态机子节 | **错误** |
| 依赖标注 | 使用缓存/MQ/RPC 的方法必须标调用目的与失败策略 | 检查下游依赖与稳定性子节 | **错误** |
| 高风险 SQL | 批量更新、无 where 更新必须标注并说明 | 检查 SQL 明细 | **错误** |

#### 6.4 图表门禁

| 检查项 | 要求 | 检查方法 | 未满足时 |
|--------|------|----------|----------|
| Mermaid 图 | 每个方法至少一个 Mermaid 图（流程图或时序图） | 检查主链路 + 时序图 | **严重错误** |
| 状态图 | 涉及状态流转的方法必须包含状态图 | 检查枚举与状态机子节 | **错误** |
| 图表一致性 | 图表必须与代码逻辑一致，不得简化关键分支 | 抽样对比代码与图表 | **严重错误** |

#### 6.5 映射门禁

| 检查项 | 要求 | 检查方法 | 未满足时 |
|--------|------|----------|----------|
| 方法 PRD 标注 | 每个方法必须标注 BR-XX 或 `TECH_METHOD` | 检查方法标识子节 | **严重错误** |
| PRD 反向覆盖 | 每个 BR-XX 必须在设计文档中有对应或 `PRD_UNMAPPED` | 检查 PRD 功能点映射表 | **严重错误** |
| 冲突入附录 | PRD 与代码冲突项必须全部入附录与质量报告 | 检查附录 CONFLICT / PRD_CODE_CONFLICT | **严重错误** |
| 未映射说明 | 未映射项必须在 quality-report.json 中标红并说明原因 | 检查质量报告 | **错误** |

#### 6.6 严格事实门禁

| 检查项 | 要求 | 检查方法 | 未满足时 |
|--------|------|----------|----------|
| EXTRACTED 占比 | 主文档正文 EXTRACTED 结论占比 = 100% | 抽样检查证据锚点 confidence | **严重错误** |
| 证据锚点 | 每个方法至少一个证据锚点，否则 BLOCKED | 检查方法块证据 | **严重错误** |
| INVALID_NO_EVIDENCE | 数量必须为 0 | 检查正文与质量报告 | **严重错误** |
| 冲突入报告 | CONFLICT / UNKNOWN_RUNTIME 必须全部入附录与质量报告 | 检查附录与 quality-report.json | **严重错误** |
| 横切能力覆盖 | 横切能力清单中「存在项」覆盖率 = 100% | 检查横切能力覆盖清单 | **错误** |

---

### 第七步：交叉引用一致性检查

#### 7.1 设计文档 ↔ PRD 一致性

**检查项**：
- 同一 BR-XX 在 PRD 中的业务规则与设计文档中的业务规则分类是否同源
- 同一 BR-XX 在 PRD 中的异常场景与设计文档中的异常链路是否同源
- 同一 BR-XX 在 PRD 中的数据需求与设计文档中的数据库表影响是否同源

**未满足时**：
- 两处以上矛盾且无 `CONFLICT` 标注 → **严重错误**

#### 7.2 设计文档 ↔ 接口文档一致性

**检查项**：
- 方法标识中的路由、HTTP Method 与接口文档是否一致
- 方法标识中的 Controller、方法名与接口文档是否一致
- 输入校验规则与接口文档中的参数校验是否一致

**未满足时**：
- 路由/方法名不一致 → **严重错误**
- 校验规则矛盾 → **错误**

#### 7.3 三角一致性

**检查项**：
- PRD 中的功能描述、设计文档中的业务说明、接口文档中的功能说明是否语义一致

**未满足时**：
- 三角矛盾 → **严重错误**

## 判定等级

| 等级 | 含义 | 处理策略 |
|------|------|----------|
| **阻断** | 无法作为设计文档使用，缺整节、示例全无、路径错误、映射完全缺失 | 必须先修复 |
| **严重错误** | 阻断交付，涉及严格事实、PRD 映射、核心链路缺失 | 必须先修复 |
| **错误** | 需修复后再进入下游，涉及表格列缺失、PRD 标注缺失、证据不足 | 必须修复 |
| **警告** | 建议修复，不阻断，涉及可读性、索引、术语、示例丰富度 | 建议修复 |

**总体结果判定规则**：
- 存在 **阻断** 或 **严重错误** → 总体结果必须为「不通过」
- 存在 **错误** 但无阻断/严重错误 → 总体结果可为「有条件通过」或「不通过」（与团队约定）
- 仅有 **警告** → 总体结果可为「通过」

## 输出格式

### 报告路径

- 保存到设计文档同目录下 `design-review/` 子目录
- 文件名：`{模块名或服务名}_设计文档完整性检查.md`
- 若检查合订多模块文档，文件名：`{服务名}_设计文档完整性检查.md`

### 报告结构

1. **检查概览**
   - 被检查文档路径
   - 检查时间
   - 总体结果（通过 / 有条件通过 / 不通过）
   - 问题计数（阻断 / 严重错误 / 错误 / 警告）

2. **文档级结构检查结果**
   - 1.1 必备文档级块
   - 1.2 PRD 覆盖清单
   - 1.3 方法索引

3. **模块级结构检查结果**
   - 2.1 模块章节边界
   - 2.2 PRD 功能点映射表

4. **方法块级完整性结果**
   - 按方法块列出 3.1~3.11 检查项结果
   - 每个方法块一节，便于定位

5. **PRD 映射一致性结果**
   - 4.1 双向可追溯
   - 4.2 命名一致性
   - 4.3 映射状态一致性

6. **严格事实模式结果**
   - 5.1 EXTRACTED 占比
   - 5.2 证据锚点完整性
   - 5.3 CONFLICT/UNKNOWN_RUNTIME 处理
   - 5.4 防发散禁令

7. **质量门禁结果**
   - 6.1 覆盖门禁
   - 6.2 链路门禁
   - 6.3 数据门禁
   - 6.4 图表门禁
   - 6.5 映射门禁
   - 6.6 严格事实门禁

8. **交叉引用一致性结果**
   - 7.1 设计文档 ↔ PRD
   - 7.2 设计文档 ↔ 接口文档
   - 7.3 三角一致性

9. **问题汇总表**
   - 编号、等级、方法块/章节、检查项、问题描述、期望修复

10. **修复建议**
    - 按等级排序，逐问题给出可执行修改建议

11. **质量报告附录**
    - 输出 `quality-report.json` 最小字段的填充结果
    - 列出 blocked_methods、invalid_conclusions、unknown_runtime_items、conflict_items
    - 列出 prd_mapping 统计
