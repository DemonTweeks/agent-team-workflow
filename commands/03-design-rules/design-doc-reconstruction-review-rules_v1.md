## 用户输入

```text
$ARGUMENTS
```

用户可提供：`02-设计文档.md` 路径、服务代码路径（Controller/Service/Repository/Mapper/Entity）、数据库 Schema/DDL 路径、（可选）`01-接口文档.md` 路径、（可选）`quality-report.json` 路径。

## 目标

- 对**单份或批量**设计返构文档执行**完整性**、**规范性**、**图表正确性**、**代码一致性**与**证据可追溯性**检查。
- 以 **「每个方法 = 独立一块（四级标题 `####`）」** 为粒度逐项核对；多模块合订时额外检查**模块业务地图**、**方法索引**。
- 输出**通过/不通过**结论、**问题清单**（阻断/主要/建议）与**可执行修改建议**。

### 文档流水线中的位置

- **上游**：`design-doc-reconstruction-rules_v1.md` 产出设计文档；依据来自后端代码（Controller、Service、Domain、Repository、Mapper、SQL、ORM 实体）。
- **交叉引用**：可对照 `01-接口文档.md` 检查方法标识与接口契约的一致性。
- **下游**：定稿设计文档作为架构评审、技术债务分析、重构规划的输入。

## 一、文档级检查（多模块合订时必检；单模块可跳过不适用项）

| 项 | 要求 | 不满足时 |
|----|------|----------|
| 服务业务域概览 | 存在服务职责、核心业务能力、技术栈、关键依赖说明 | **主要** |
| 模块业务地图 | 存在模块清单、职责简述、模块间依赖关系图（文字描述或 Mermaid） | **主要** |
| 方法索引 | 存在表格（序号、模块、Controller#method、中文名、涉及表、事务、异步调用），便于跳转 | **主要** |
| 修订记录 | 版本、日期、作者、变更摘要（可选但推荐） | 缺失 → **建议** |
| 与接口文档交叉引用 | 方法索引或方法标识中引用对应 `01-接口文档.md` 章节 | **建议** |

---

## 二、模块级检查（每个 `##` 模块块必检）

### 2.1 模块边界

- 每个模块对应**一个二级标题** `## {序号}. {模块名}`。
- 模块块内必须包含：

| 必选子节 | 最低要求 | 缺失时 |
|----------|----------|--------|
| 模块职责与边界 | 职责、边界、关键实体、与相邻模块的分界 | **主要** |
| 方法级业务链路 | 至少一个方法级设计块 | **阻断** |
| 模块规则清单汇总 | 该模块全部方法的规则分类汇总 | **主要** |
| 模块枚举与状态机总表 | 该模块全部枚举与状态机汇总 | **建议** |
| 模块数据库表影响总表 | 该模块全部表影响汇总 | **主要** |

### 2.2 模块职责与边界

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| 职责描述 | 一句话清晰描述该模块的业务边界 | **主要** |
| 边界说明 | 与相邻模块的职责分界 | **建议** |
| 关键实体 | 该模块涉及的核心领域实体 | **建议** |
| Controller 列表 | 该模块包含的 Controller 清单 | **建议** |
| 路由前缀 | 该模块的路由前缀 | **建议** |

---

## 三、按方法块的完整性检查（每个 `####` 方法块必检）

### 3.1 块边界

- 每个业务方法对应**一个四级标题** `#### {序号}. {Controller#method} — {方法中文名称}`。
- 块内必选子节（标题语义一致即可，允许「1. 方法标识」等形式）：

| 必选子节 | 最低要求 | 缺失时 |
|----------|----------|--------|
| **方法标识** | 模块名、Controller、方法名、路由、接口文档引用、业务说明 | **阻断**或**主要** |
| **主链路（流程图）** | Mermaid flowchart，覆盖主干流程，标注事务边界 | **阻断** |
| **时序图** | Mermaid sequenceDiagram，覆盖对象交互，标注异步与异常 | **阻断** |
| **分支链路** | 表格列出所有条件分支 | **主要** |
| **异常链路** | 表格列出所有异常处理路径 | **主要** |
| **业务规则分类** | 按 8 类强制分类汇总（准入/校验/流转/并发/幂等/时效/审计/补偿） | **主要** |
| **枚举与状态机** | 枚举清单；如涉及状态机须有状态图或流转矩阵 | **主要** |
| **数据库与表影响** | 方法级表影响表、SQL 明细（高风险操作）、表级汇总 | **主要** |
| **下游依赖与稳定性** | 表格列出 HTTP/RPC/MQ/Cache/DB 依赖及失败策略 | **主要** |
| **方法链路追踪** | 文本级调用树，便于快速定位代码 | **建议** |

### 3.2 方法标识

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| 模块名 | 与上级模块一致 | **主要** |
| Controller | 类全名或简名 | **建议** |
| 方法名 | 与代码一致 | **主要** |
| 路由 | HTTP Method + Path | **主要** |
| 接口文档引用 | 引用对应 `01-接口文档.md` 章节锚点 | **建议** |
| 业务说明 | 一句话该方法在业务上的职责 | **主要** |

### 3.3 主链路（流程图）检查

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| Mermaid 语法 | 必须为有效 `flowchart TD` 或 `flowchart LR` | **阻断** |
| 节点覆盖 | 输入校验 → 权限校验 → 业务编排 → 数据操作 → 对外调用 → 返回组装 | **主要** |
| 代码位置标注 | 每个节点标注类名#方法名 | **主要** |
| 条件判断 | 菱形节点，标注判断条件 | **建议** |
| 循环/批量 | 用子图包裹 | **建议** |
| 事务边界 | 虚线框标注 TX 开始/结束/回滚 | **主要** |
| 与代码一致 | 流程图节点须与代码实际调用链一致 | **阻断** |

### 3.4 时序图检查

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| Mermaid 语法 | 必须为有效 `sequenceDiagram` | **阻断** |
| 参与者完整 | 须含：调用方、Controller、Service、Domain、Repository、DB、外部服务/MQ/缓存 | **主要** |
| 消息标注 | 须标注方法名和关键参数 | **主要** |
| 异步调用 | 虚线箭头 `-->>` | **建议** |
| 事务边界 | `rect` 包裹并标注传播行为 | **主要** |
| 异常返回 | 红色标注或 `Note over X: Exception` | **建议** |
| 自调用 | 须显式展示 | **建议** |
| 与代码一致 | 时序须与代码实际调用顺序一致 | **阻断** |

### 3.5 分支链路检查

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| 表格完整 | 序号、分支条件、触发阈值/条件表达式、分支动作、分支返回、代码位置 | **主要** |
| 代码位置 | 每个分支须标注类名#方法名或行号范围 | **主要** |
| 条件具体 | 复杂分支条件须展开为子表或伪代码 | **建议** |
| 策略模式 | 如使用策略模式须列出全部策略实现类 | **建议** |
| 与代码一致 | 分支条件须与代码中的 if/switch 一致 | **主要** |

### 3.6 异常链路检查

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| 表格完整 | 异常类型、抛出点、捕获点、转换规则、返回码/HTTP 状态、日志/告警、代码位置 | **主要** |
| 业务/系统区分 | 须区分业务异常（预期内）与系统异常（预期外） | **建议** |
| 事务回滚 | 须标注异常是否触发事务回滚 | **主要** |
| 降级/兜底 | 须标注异常是否触发降级/兜底逻辑 | **建议** |
| 全局处理器 | 全局异常处理器须单独说明 | **建议** |
| 与代码一致 | 异常类型与代码中的 throw/catch 一致 | **主要** |

### 3.7 业务规则分类检查

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| 8 类强制分类 | 准入/校验/流转/并发/幂等/时效/审计/补偿 | **主要** |
| 规则具体 | 禁止泛泛而谈，须写清「校验了xx字段的yy规则」 | **主要** |
| 证据锚点 | 每个规则须附证据锚点（file_path + symbol） | **主要** |
| 触发条件 | 须标注规则触发条件 | **建议** |
| 失败处理 | 须标注规则失败后的处理 | **建议** |
| 与代码一致 | 规则须与代码中的校验/判断逻辑一致 | **主要** |

### 3.8 枚举与状态机检查

#### 3.8.1 枚举清单

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| 表格完整 | 枚举名、所属类、用途、全部值与含义、代码位置 | **主要** |
| 值完整 | 须列出全部枚举值，不得遗漏 | **主要** |
| 含义清晰 | 每个值须有业务语义说明 | **主要** |
| 与代码一致 | 枚举值与代码定义一致 | **阻断** |

#### 3.8.2 状态机

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| Mermaid 状态图 | 如涉及状态机，须有有效 `stateDiagram-v2` | **主要** |
| 状态集合 | 须列出全部状态 | **主要** |
| 流转矩阵 | 表格：当前状态、允许下一状态、触发事件、守卫条件、失败回滚策略、代码位置 | **主要** |
| 与代码一致 | 状态流转须与代码中的条件判断一致，不得臆测 | **阻断** |
| 工作流引擎 | 如涉及 Activiti/Camunda 等须说明流程定义文件位置 | **建议** |

### 3.9 数据库与表影响检查

#### 3.9.1 方法级表影响

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| 表格完整 | database、schema、table、operation、where_keys、write_fields、join_tables、index_or_unique_dependency、transaction_scope、代码位置 | **主要** |
| operation 准确 | SELECT/INSERT/UPDATE/DELETE/UPSERT | **主要** |
| 事务边界 | 须标注事务范围 | **主要** |
| 与代码一致 | 表名、字段名、操作类型须与 SQL/ORM 一致 | **阻断** |

#### 3.9.2 SQL 明细

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| 高风险操作 | 复杂查询、批量更新、无 where 更新须展开 SQL 或伪 SQL | **主要** |
| SQL 准确 | SQL 须与 Mapper XML / 原生 SQL / ORM 生成 SQL 一致 | **阻断** |
| 乐观锁 | 如涉及乐观锁须展示 version 条件 | **建议** |

#### 3.9.3 表级汇总

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| 表格完整 | 表名、业务用途、上游接口、核心字段语义、逻辑删除字段、乐观锁字段、高风险操作 | **主要** |
| 多库/多租户 | 多库、多租户场景须标注数据源配置 | **建议** |
| 无证据处理 | 无 SQL/ORM 证据时须写 `UNKNOWN_RUNTIME`，不得臆测表关系 | **主要** |

### 3.10 下游依赖与稳定性检查

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| 表格完整 | 依赖类型、依赖目标、调用目的、超时设置、重试策略、熔断/降级、补偿机制、代码位置 | **主要** |
| 失败策略 | 每个依赖须标注失败后的处理策略 | **主要** |
| 异步可靠性 | 异步调用须标注消息可靠性 | **建议** |
| 缓存策略 | 缓存须标注读写策略与失效策略 | **建议** |
| 与代码一致 | 依赖目标、超时、重试配置须与代码一致 | **主要** |

### 3.11 方法链路追踪检查

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| 文本调用树 | 主链路 + 分支链路 + 异常链路的文本级调用树 | **建议** |
| 节点标注 | 每个节点须标注类型（Controller/Service/Domain/Repository/DB/External） | **建议** |
| 特殊标记 | 须标注 TX/RPC/MQ/Cache/分支/异常 | **建议** |
| 与代码一致 | 调用树须与代码实际调用链一致 | **主要** |

## 四、图表质量检查

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| Mermaid 图数量 | 每个方法至少 1 个 Mermaid 图（流程图或时序图） | **阻断** |
| 状态机图 | 涉及状态流转的方法必须包含状态图 | **主要** |
| 图表与代码一致 | 图表节点、消息、条件须与代码逻辑一致，不得简化关键分支 | **阻断** |
| 图表可读性 | 节点命名清晰、箭头方向合理、不重叠 | **建议** |
| 事务标注 | 流程图与时序图均须标注事务边界 | **主要** |
| 异步标注 | 异步调用须在时序图中用虚线箭头标注 | **建议** |

## 五、横切能力覆盖检查

以下横切能力必须在文档中至少出现一次（有则检查质量，无则须标 N/A）：

| 能力维度 | 检查点 | 文档位置 | 未覆盖时 |
|----------|--------|----------|----------|
| 认证与授权 | JWT、RBAC、ABAC 实现方式 | 准入规则/模块规则清单 | **建议** |
| 租户隔离与数据权限 | 租户字段、数据权限过滤逻辑 | 数据库与表影响/准入规则 | **建议** |
| 限流与防刷 | 限流规则、防刷策略 | 非功能特征/模块规则清单 | **建议** |
| 幂等与防重 | 幂等键、窗口期、去重逻辑 | 幂等规则/下游依赖 | **建议** |
| 事务传播与最终一致性 | 事务边界、传播行为、补偿机制 | 主链路/异常链路 | **主要** |
| 缓存策略 | 旁路/写透/双删、失效策略 | 下游依赖与稳定性 | **建议** |
| 消息可靠性 | 投递确认、重试、死信、消费幂等 | 下游依赖与稳定性 | **建议** |
| 异常分层 | 业务异常/系统异常定义与处理 | 异常链路 | **主要** |
| 降级熔断 | 本地兜底/远程兜底、熔断条件 | 下游依赖与稳定性 | **建议** |
| 可观测性 | 日志字段标准、traceId、指标埋点 | 方法链路/异常链路 | **建议** |
| 安全 | 脱敏、加密、签名、防重放 | 业务规则分类/模块规则清单 | **建议** |
| 配置治理 | 动态开关、灰度、阈值配置 | 分支链路/模块规则清单 | **建议** |
| 批处理/定时任务 | 批处理逻辑、定时任务触发 | 模块规则清单 | **建议** |
| 并发控制 | 锁（分布式/本地）、乐观锁、唯一约束 | 并发规则/数据库与表影响 | **主要** |
| 版本兼容 | 接口版本、字段兼容策略 | 方法标识/分支链路 | **建议** |

## 六、证据可追溯性检查

| 检查项 | 要求 | 不满足时 |
|--------|------|----------|
| 证据锚点 | 每个方法至少 1 个证据锚点（file_path + symbol + evidence_type） | **主要** |
| 证据类型完整 | evidence_type 须为：route/validation/auth/sql/transaction/enum/cache/mq/rpc/exception/state_machine 之一 | **建议** |
| 置信度 | 主文档正文只含 EXTRACTED 结论 | **主要** |
| INFERRED/AMBIGUOUS | 仅出现在「候选/待确认」区块，不得混入正文 | **主要** |
| UNKNOWN_RUNTIME | 未观察到证据的内容必须标记 UNKNOWN_RUNTIME | **建议** |
| 无证据推断 | 禁止无证据推断，否则标记 INVALID_NO_EVIDENCE | **阻断** |
| 冲突处理 | 证据冲突时须标记 CONFLICT 并并列展示，不得自行合并 | **主要** |

|--------|------|----------|

| 分支条件 | 分支链路表格与代码 if/switch 条件一致 | **主要** |
| 异常类型 | 异常链路表格与代码 throw/catch 一致 | **主要** |

| 事务边界 | 事务标注与代码 @Transactional 一致 | **主要** |

## 九、严重级别

| 级别 | 含义 | 示例 |
|------|------|------|
| 阻断 | 无法作为设计参考；缺整节、图表错误、调用链与代码不符、无证据推断 | 缺主链路/时序图；Mermaid 语法错误；表名与 SQL 不一致；Controller 到 Service 链路缺失 |
| 主要 | 缺子节、缺表、图表与代码不一致、字段描述质量不达标、证据缺失 | 缺分支/异常链路；状态流转与代码不符；业务规则无证据锚点；数据库影响缺失 |
